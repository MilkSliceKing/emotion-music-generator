# 项目设计说明书

## 一、项目概述

**名称**：情绪音乐生成器 (Emotion Music Generator)

**目标**：通过摄像头实时捕捉用户面部表情，识别情绪状态（Happy / Sad / Angry / Surprised / Neutral / Fear / Disgust），并动态生成对应风格的钢琴音乐，实现"表情-音乐"的实时互动体验。

**技术栈**：C++17 + OpenCV + dlib + onnxruntime + CMake

**运行环境**：Linux (Ubuntu)

---

## 二、系统架构

```
摄像头/图片/视频输入 → 面部检测 → 关键点提取 → 情绪识别 → 情绪映射 → 音乐生成 → 音频播放
     (OpenCV)        (dlib HOG)  (dlib 68pt)  (ONNX)    (查表)   (算法)   (paplay)
```

### 模块职责

| 模块 | 文件 | 职责 |
|------|------|------|
| 面部检测 | `detector/face_detector` | 使用 dlib HOG+SVM 检测人脸，提取 68 关键点 |
| 情绪识别 | `detector/emotion_recognizer` | HSEmotion ONNX 模型 + onnxruntime 推理，7 类情绪 + softmax 置信度 |
| 情绪映射 | `mapper/emotion_mapper` | 将情绪转换为音乐参数（调性、节拍、力度、和弦进行、伴奏模式） |
| 音乐生成 | `generator/music_generator` | 和弦进行 + 旋律生成 + 左手伴奏 + 乐句结构 |
| 音频播放 | `audio/audio_player` | WAV 波形合成（钢琴音色模拟）+ paplay/aplay 播放 |
| UI 渲染 | `ui/overlay_renderer` | 毛玻璃面板 + 置信度条形图 + 情绪历史曲线 + 均衡器动画 + 屏幕按钮 |

## 三、核心算法

### 3.1 人脸检测（dlib HOG + 68 关键点）

使用 dlib 的 `frontal_face_detector`（HOG + SVM 级联）检测正脸，再用 `shape_predictor_68_face_landmarks` 提取 68 个面部关键点。

| 区域 | 索引 | 用途 |
|------|------|------|
| 下巴轮廓 | 0-16 | 人脸边界 |
| 左眉 | 17-21 | 眉毛高度特征 |
| 右眉 | 22-26 | 眉毛高度特征 |
| 鼻梁/鼻翼 | 27-35 | 面部朝向参考 |
| 左眼 | 36-41 | 眼睛开合度 |
| 右眼 | 42-47 | 眼睛开合度 |
| 外唇/内唇 | 48-67 | 嘴巴张合、嘴角角度 |

### 3.2 情绪识别（HSEmotion ONNX + onnxruntime）

**模型**：HSEmotion EfficientNet-B0（ICML 2023，HSE 大学 + Sber AI Lab）

**推理流程**：
1. 裁剪人脸区域 + 20% 边距
2. 缩放到 224×224，BGR → RGB
3. 归一化：`(pixel/255 - mean) / std`（ImageNet mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225]）
4. 构造 NCHW tensor [1,3,224,224]
5. onnxruntime 推理，得到 8 类 logits
6. Softmax 转概率，映射 8 类 → 7 类（Contempt 归入 Neutral）

### 3.3 情绪映射

7 种情绪各自映射到独立的音乐参数组合：

| 情绪 | 调性 | 节拍 | 力度 | 和弦进行 | 伴奏模式 |
|------|------|------|------|---------|---------|
| Happy | C Major | 120 BPM | bright | I-V-vi-IV | 分解和弦 |
| Sad | A Minor | 70 BPM | melancholic | i-VII-VI-VII | 分解和弦 |
| Angry | E Minor | 140 BPM | intense | i-iv-V-i | 柱式和弦 |
| Surprised | G Major | 130 BPM | playful | I-vi-IV-V | 阿尔贝蒂低音 |
| Neutral | C Major | 100 BPM | calm | I-IV-V-I | 分解和弦 |
| Fear | D Minor | 80 BPM | tense | i-VI-III-VII | 分解和弦 |
| Disgust | F Minor | 90 BPM | dark | i-iv-i-iv | 柱式和弦 |

### 3.4 音乐生成

**和弦进行系统**：罗马数字字符串解析为具体 MIDI 音高的和弦序列，大/小调使用不同音阶度数映射。

**旋律生成**：
- 强拍：70% 和弦音，30% 音阶音
- 弱拍：40% 和弦音，60% 经过音
- 乐句结构：4 小节一句，力度曲线 [0.6, 0.8, 1.0, 0.7]（起承转合）
- 结尾 V-I 解决

**左手伴奏**（三种模式，低一个八度）：
- 分解和弦：根音→三度→五度→根音
- 柱式和弦：每小节第 1 拍同时弹三个音
- 阿尔贝蒂低音：根音→五度→三度→五度快速交替

### 3.5 音频合成

**钢琴音色模拟**：
- 双段衰减包络：快速初始衰减 + 慢速尾音（模拟真实钢琴弦振动）
- 三弦模拟：3 个微调振荡器（detune ±0.06%），模拟拍频效应
- Schroeder 混响：4 comb + 2 allpass 滤波器
- 击弦瞬态噪声脉冲
- 非谐性模型（stretched tuning）
- 琴体共鸣：低频泛音增强
- 力度非线性响应：`pow(vel_factor, 1.5)`

**混合播放**：`playComposition()` 将旋律和伴奏叠加到同一时间线上，全局归一化防削波。

## 四、数据流

```
cv::Mat (输入帧)
    ↓
FaceDetector::detectFaces() → vector<FaceRect>
    ↓
FaceDetector::getLandmarks() → vector<cv::Point> (68 点)
    ↓
EmotionRecognizer::recognizeFromImage() → Emotion (枚举) + confidences[7]
    ↓
EmotionMapper::mapToMusic() → MusicParams {key, scale, tempo, velocity, mood, progression, accompaniment}
    ↓
MusicGenerator::generateComposition() → Composition {vector<Note>, total_duration}
    ↓
AudioPlayer::playComposition() → 波形合成 → WAV → paplay 播放
```

## 六、设计原则

1. **先跑通再优化**：从规则判断 → DeepFace 桥接 → ONNX 纯 C++ 推理，逐步迭代
2. **模块化**：每个模块独立编译，通过清晰接口交互，CMakeLists.txt 体现结构
3. **数据驱动**：关键参数通过 config.yaml 管理，避免硬编码
4. **编译优化**：CMakeLists.txt 区分 Debug/Release，Release 启用 -O3 -march=native
