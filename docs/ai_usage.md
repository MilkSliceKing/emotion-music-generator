### AI 生成代码占比

| 模块 | AI生成比例 | 说明 |
|------|-----------|------|
| face_detector | ~80% | dlib API 使用由AI生成，调试由人工完成 |
| emotion_recognizer | ~90% | 特征提取算法和阈值由AI设计 |
| emotion_mapper | ~95% | 简单映射表 |
| music_generator | ~90% | 音阶定义和旋律生成策略 |
| audio_player | ~90% | PortAudio API 使用 |
| main.cpp | ~85% | 主循环框架由AI生成 |
| CMakeLists.txt | ~90% | 构建配置 |
| 文档 | ~80% | 框架由AI生成，实际数据由人工填写 |

---

### 使用体会

- **优点**：AI 能快速生成项目骨架和 API 调用代码，显著加速开发
- **局限**：AI 生成的阈值参数（情绪识别）需要在实际环境中调优
- **注意**：AI 生成的代码需要人工审查和测试，不能直接信任

## 示例：与 AI 的交互

以下为使用AI辅助开发的典型交互示例：

```
用户: 请帮我设计情绪识别的算法，基于dlib的68关键点

AI: 基于dlib 68关键点，可以从以下特征进行情绪分类：
    1. 嘴巴纵横比(MAR) — 反映嘴巴张开程度
    2. 眼睛纵横比(EAR) — 反映眼睛睁大程度
    3. 眉毛高度 — 反映眉毛上扬或下压
    4. 嘴角角度 — 反映微笑或悲伤

    基于这些特征设置阈值，按照优先级进行规则判断...
```

（以上为示例，实际交互记录可在 Claude Code 会话历史中查看）
